[A Basic Diplomacy]

solution

题面：每天有若干个数可以选择，共 $m$ 天。要求每天选出一个数，使得出现次数最多的数的数量不超过 $\lceil\frac{m}{2}\rceil$。

做法：容易注意到能超过的最多一个数，于是除却每天只能选一个的情况，其它的我只要不选会超过的就一定合法了。

[B Playlist]

solution

题面：$n$ 个数成环，按序每次选择一对相邻的数，若 $\gcd=1$ 则删去后面的那个，然后继续往后选。问最终剩下的数和删数的顺序。

做法：用一个 set 维护相邻 $\gcd=1$ 的数，然后每次从当前的数找后继即可，删去后像链表一样更新相邻的即可。时间复杂度 $O(n\log n)$。

[C Skyline Photo]

solution

题面：$n$ 个数有高度和权值，将序列分成若干段区间，每个区间的权值为最矮的数的权值。求最大的分割权值。

做法：显然有个 dp 是设 $f_i$ 表示最后一个数为 $i$ 的最大权值。转移就是 $f_i=\max\lbrace f_{j}+w(j+1,i)\rbrace$，这东西显然的单调栈维护。时间复杂度 $O(n)$。

[D Useful Edges]

solution

题面：一张无向图，有边权。若干条限制 $(u,v,w)$，表示所有 $u$ 到 $v$ 边权和小于等于 $w$ 的路径上的边。问限制并的大小。

做法：如果算单个我就不会了，算并就很简单了。

考虑一条边 $(i,j,k)$ 合法当且仅当存在一个 $(u,v,w)$，使得 $dis(u,i)+k+dis(j,v)\leq w$，其中 $dis$ 表示两点的最短路。注意到 $w$ 实际上只是给 $u,v$ 增加限制，于是可将 $w$ 写成 $l(u,v)$。于是又可以改写成 $dis(u,i)+k\leq l(u,v)-dis(j,v)$。注意到左边是 $O(n^3)$ 的信息量，右边也是 $O(n^3)$。于是右边预处理出来，枚举左边就做完了。时间复杂度 $O(n^3)$。

[E Vabank]

solution

题面：交互，未知一个数 $M$，初始有一个数 $X$，你每次可以猜一个数 $Y$，若 $Y\leq M$，则 $X\leftarrow X+Y$；若 $Y>M$，则 $X\leftarrow X-Y$。若 $X<0$ 则算失败。$M\in [0,10^{14}]$，交互次数为 $105$。

做法：首先不断倍增，用 $47$ 次机会就可以得到 $M\in [2^i,2^{i+1})$。然后我们在这个范围内二分，这样次数大概是 $130$ 次，过不了。

接下来分为两条路子，一条是直接调整二分的 $mid$。注意到二分时很多次数的使用都是为了让 $X$ 到 $mid$，所以如果贪心一点，然后二分的 $mid$ 尽量接近 $r$ 的话，就很可能可以用更少的次数成功了。这个调整 $mid$ 是题解的做法，但我没看，不过自己随便调下参似乎就过了。还有一条路子是考虑 dp 出最坏情况，设 $f_{i,j}$ 表示还剩 $i$ 次机会，$j$ 次减法机会的最坏情况下最多减少的区间数。这个东西是可以直接递推的，即 $f_{i,j}=f_{i-1,j-1}+f_{i-1,j+1}$。根据这个也可以轻松调整 $mid$ 来完成此题。

代码写的前者，调的参数在代码里。

[F Exam]

solution

题面：给定互不相同的字符串 $s_1, s_2, \cdots, s_n$，求有多少对 $(i, j)$ 满足：

  • $i \neq j$
  • $s_j$ 是 $s_i$ 的子串。
  • 不存在 $k$ $(k \neq i, k \neq j)$ 满足 $s_j$ 是 $s_k$ 的子串且 $s_k$ 是 $s_i$ 的子串。

保证 $n, \sum \lvert s_i \rvert \leq 10^6$。

做法：考虑枚举 $s_i$，然后找 $s_j$。一个合法的 $s_j$ 要求它出现在 $s_i$ 的所有位置处都不被其他字符串覆盖，这个东西直接是不太好维护的。换个思路，我们对每个 $s_i$ 的每个前缀找到其的最长后缀满足它是这若干个字符串中的一个，那么如果 $s_j$ 是合法的，则它出现的每个位置所对应的前缀所能找到的最长后缀一定就是 $s_j$，且不被包含。这样子找的位置数就是 $O(\sum |s_i|)$。换句话说，一个字符串是最长后缀的次数等于出现次数且后面找到的最靠前的后缀没有包含它时，这个字符串就是合法的。而这些信息全部可以用 $\mathrm{ACAM}$ 维护，最长后缀就是不断跳 $\mathrm{fail}$ 所到达的第一个字符串位置，这个直接 dfs 就预处理出来了，出现次数就是多个子树和，用 $\mathrm{dfn}$ 序与 $\mathrm{BIT}$ 即可。时间复杂度 $O(\sum |s_i|\log)$。
